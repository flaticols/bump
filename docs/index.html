Bump

Semver Tag Management for Git Repositories

About Installation Usage Examples Contribute GitHub

About

Bump is a lightweight command-line tool for managing semantic versioning tags in Git repositories. It automates version increments following semver standards, making it easy to maintain proper versioning in your projects.

With Bump, you can quickly increment major, minor, or patch versions with simple commands, streamlining your release process.

Installation

Using Go

$ go install github.com/flaticols/bump

Using Homebrew

$ brew tap flaticols/tap
$ brew install bump

From Source

$ git clone https://github.com/flaticols/bump.git
$ cd bump
$ make install
Download Binary

Usage

Basic syntax:

$ bump [command] [options]

Commands

Major Version

Increment the major version (for breaking changes)

$ bump major
# e.g., 1.2.3 → 2.0.0

Minor Version

Increment the minor version (for new features)

$ bump minor
# e.g., 1.2.3 → 1.3.0

Patch Version

Increment the patch version (for bug fixes)

$ bump patch
# e.g., 1.2.3 → 1.2.4

Current Version

Show the current version

$ bump current
v1.2.3

Examples

Creating a pre-release version

$ bump minor --pre alpha
# e.g., 1.2.3 → 1.3.0-alpha.1

Adding a tag message

$ bump patch -m "Fix critical bug in authentication"
# Creates an annotated tag with the message

Creating a custom version

$ bump set 2.5.0
# Sets version to exactly 2.5.0

Viewing version history

$ bump history
v1.2.4 - 2023-06-15
v1.2.3 - 2023-06-10
v1.2.2 - 2023-06-05
v1.1.0 - 2023-05-20
v1.0.0 - 2023-05-01
